    冒泡排序算法比较简单，假设待排序列的长度为n，其思想是：

    什么是tmux

    tmux是一个终端的复用器（terminal multiplexer），可以简单理解为一个终端的窗口管理程序。相比系统自带的 screen 功能要强大许多。其使用场景和特性包括：

    • 当你使用SSH连接到服务器的时候，由于网络掉线导致连接中断（这是经常发生的。。。）。当你重新连接后，所有的工作现场丢失了。你还得重新打开之前的各种窗口，进入各种很深的目录，非常麻烦。而如果使用tmux，即时掉线之后，重新连接也能直接回到之前的工作环境，极大提高工作的效率。
    • 可以完全通过键盘操控
    • 支持自定义的按键绑定模式，例如设置成vim按键绑定模式
    • 可以在一个SSH连接下打开多个会话、窗口，也可以对窗口进行各种分割，效果如下图所示。

    建议50：利用模块设计实现单例模式

    GOF的23种设计模式中，单例是最常用的模式，通过单例模式可以保证系统中一个类只有一个实例而且该实例易于被外界访问，从而方便对实例个数的控制并节约系统资源。

    Python语言实现实例没有那么方便，因为缺乏声明私有构造函数的语法元素，实例又带有类型信息。如：

    class _Singleton(object):
    pass
Singleton = _Singleton()
del _Singleton
another = Singleton.__class__()
print type(another)

# 输出 <class '__main__._Singleton'>

    虽然把Singleton的类定义删除了，但是还是可以通过已有实例的__class__属性生成一个新的实例。

    • 所有变量都会绑定到模块
    • 模块只初始化一次
    • import 机制是线程安全的
