  2. 终端窗口管理神器——tmux

    Mon 11 May 2015
    By wwt

    什么是tmux

    tmux是一个终端的复用器（terminal multiplexer），可以简单理解为一个终端的窗口管理程序。相比系统自带的 screen 功能要强大许多。其使用场景和特性包括：

    • 当你使用SSH连接到服务器的时候，由于网络掉线导致连接中断（这是经常发生的。。。）。当你重新连接后，所有的工作现场丢失了。你还得重新打开之前的各种窗口，进入各种很深的目录，非常麻烦。而如果使用tmux，即时掉线之后，重新连接也能直接回到之前的工作环境，极大提高工作的效率。
    • 可以完全通过键盘操控
    • 支持自定义的按键绑定模式，例如设置成vim按键绑定模式
    • 可以在一个SSH连接下打开多个会话、窗口，也可以对窗口进行各种分割，效果如下图所示。

    tmux的基本概念

    首先，需要先安装tmux：

    sudo yum install tmux

    安装好后，直接输入 tmux 即可运行 tmux.

    • session：会话，是一组窗口的集合，通常同一个任务可以放入一个session。可以同时打开多个session，每个session可以有自己的名字方便任务的切换。

    3. 可视模式：按下键v，进入可视模式，可以移动光标对文本进行选择

    静态网站

    早在多年之前，我就使用过WordPress + MySQL + Apache在国外的主机上搭建过个人博客。但是，对于个人博客而言，使用WordPress这种动态网站，还需要后台支持各种数据库，就显得过于臃肿。最近几年人们纷纷转向静态博客，加之Github Pages和Dropbox这些地方可以免费的放置这些静态页面，于是各种静态网站生成器便如雨后春笋出现，不完全统计的就有392种之多，这里由列出了所有的Static site generator 列表及 Github 关注度排名。

    什么是静态网站

    静态的网站全部由静态网页构成，这些网页都是提前生成好的，不需要与后台数据库进行交交互，并且内容不会动态改变。静态网站具有如下特点：

    • 不需要数据库

    • 访问速度更快

    • 基本没有安全性的问题

    • 使写作者更加专注于内容本身
