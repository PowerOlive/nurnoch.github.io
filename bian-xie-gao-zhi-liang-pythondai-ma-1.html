编写高质量Python代码（1）

Mon 06 April 2015
By wwt

去年参加了珠海的PyCon大会，幸运地抽中了三本书，其中一本是《编写高质量代码：改善Python程序的91个建议》。本书从多个方面给出了编写高质量Python代码的91个建议，我把每一章的建议都先总结下来，分享给大家。

建议1： 理解Pythonic概念

（1）Pythonic的定义

遵循Pythonic的代码看起来像伪代码：

def quicksort(array):
    less = []
    greater = []
    if len(array) <= 1:
        return array
    pivot = array.pop()
    for x in array:
        if x > pivot:
            greater.append(x)
        else:
            less.append(x)
    return quicksort(less) + [pivot] + quicksort(greater)
if __name__ == '__main__':
    print quicksort([3,2,4,5,6,7,8])

可读性比伪代码还要好！

（2）代码风格

Pythonic的代码要充分体现Python自身特色。 例如交换两个数(packaging/unpackaging)：

a = 1
b = 2
a, b = b, a

还有需要安全关闭文件描述符，可以使用with语句：

with open(path,'r') as f:
    do_st(f)

我们应当追求充分利用Python的语法，但不应过分追求奇技淫巧，比如利用Slice语法：

a = [1,2,3,4]
c = 'abcdef'
print a[::-1]
print c[::-1]

这是一个实现列表倒序输出的功能，但是如果不熟悉该语法的可能就看不懂，实际上，这个时候更能体现Pythonic的代码是充分利用Python的库里的reversed函数：

list(reversed(a))
[4, 3, 2, 1]

（3）标准库

写Pythonic的程序必须对Python的标准库有充分了解，特别是内置函数与内置数据类型。 例如格式化输出字符串,通常的做法：

print 'Hello %s' % ('Tom',)

Pythonic的代码：

print 'Hello %{name}s' % {'name': 'Tom'}

尤其是参数多的时候可读性更好。 str.format()是Python最为推荐的字符串格式化方法，当然也是最Pythonic的。

print '{greet} from {language}.'.format(greet = 'Hello world', language = 'Python')

（4）Python的库或框架

公认Flask这个框架式比较Pythonic的。可以仔细阅读这些优秀Python开源项目的源码。

建议2： 编写Pythonic代码

（1）避免劣化代码

通常有几个需要注意：

  • 避免只用大小写来区分不同的对象。如a和A
  • 避免使用容易引起混淆的名称
  • 不要害怕过长的变量名，如person_info比pi可读性强得多。

（2）深入认识Python有助于编写Pythonic代码

  • 看官方Reference
  • 学习新版本特性
  • 深入学习公认的Pythonic的代码，比如Flask、gevent和requests等。

（3）使用工具检查程序PEP8

安装PEP8的检测程序

pip install -U pep8

下面使用工具检查：

if __name__ == '__main__':
    print quicksort([3,2,4,5,6,7,8])

可以看到检查的结果：

pep8 --first quick_sort.py
quick_sort.py:17:23: E231 missing whitespace after ','
quick_sort.py:18:1: W391 blank line at end of file

就是，后面要空格，文件最后不应该有空行。 还可以展示出非常详细的错误和警告：

pep8 --show-source --show-pep8 quick_sort.py

PEP8也并非是唯一的规范，还有Google Python Style Guide等等。

建议3： 理解Python与C语言的不同之处

Python底层是使用C来写的，但不要把C语言的思维方式带入Python。 可以利用Python实现C中Python并不自带的功能：

三元运算符： C ? X : Y => X if C else Y

类似switch的跳转功能：

def f(x):
    return {
            0: "You typed zero.\n",
            1: "You are in top.\n"
            }.get(x, "Only single-digit numbers are allowed\n")
if __name__ == '__main__':
    print f(1)

建议4： 在代码中适当注释

Python中有3种形式的注释：块注释、行注释和文档注释（docstring）。 注释和代码隔开一定的距离； 给外部访问的函数和方法添加文档注释，包括方法功能，参数，返回值以及可能的异常进行说明。

建议5： 通过适当添加空行使代码布局更为优雅、合理

如以下代码：

import random
guess_made = 0
name = raw_input('Hello! What is your name?\n')
number = random.randint(1, 20)
print 'Well, {0}, I am thinking of a number between 1 and 20.'\
        .format(name)
while guess_made < 6:
    guess = int(raw_input('Take a guess: '))
    guess_made += 1
    if guess < number:
        print 'Your guess is too low.'
    if guess > number:
        print 'Your guess is too high.'
    if guess == number:
        break
if guess == number:
    print 'Good job'
else:
    print 'Nope'
  • 一组代码表达完一个完整的思路后，应该用空白行隔开
  • 尽量保持上下文语义的易理解性。如当一个函数调用另一个函数时，尽量将它们放在一起，最好调用者在上，被调用者在下。
def A():
    B()
def B():
    pass
  • 避免过长的代码行。超过部分使用括号进行连接
  • 逗号和分号之前不要空格；二元运算符左右应该有空格；
  • 函数名和左括号不需要空格

建议6： 编写函数的4个原则

  • 函数设计尽量短小，嵌套层次不宜过深。尽量不要太长，需要来回滚动。
  • 函数声明做到合理、简单、易于使用
  • 函数设计考虑向下兼容，如加入默认参数避免退化，做到向下兼容。
  • 一个函数只做一件事！

建议7：将常量集中到一个文件中

  • 通过命名风格提醒使用者该变量代表为常量，如使用大写字母加下划线。
  • 通过自定义的类实现常量功能。
