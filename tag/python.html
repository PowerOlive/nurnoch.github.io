    建议50：利用模块设计实现单例模式

    GOF的23种设计模式中，单例是最常用的模式，通过单例模式可以保证系统中一个类只有一个实例而且该实例易于被外界访问，从而方便对实例个数的控制并节约系统资源。

    Python语言实现实例没有那么方便，因为缺乏声明私有构造函数的语法元素，实例又带有类型信息。如：

    class _Singleton(object):
    pass
Singleton = _Singleton()
del _Singleton
another = Singleton.__class__()
print type(another)

# 输出 <class '__main__._Singleton'>

    虽然把Singleton的类定义删除了，但是还是可以通过已有实例的__class__属性生成一个新的实例。

    其实模块采用的方法是天然的单例实现方式：

    • 所有变量都会绑定到模块
    • 模块只初始化一次
    • import 机制是线程安全的

  4. Python容器——Counter的使用

    Wed 08 April 2015
    By wwt

    Counter是Python标准库提供的一个非常有用的容器，可以用来对序列中出现的各个元素，进行计数。这篇文章中，利用Counter来实现多元集合(MultiSets)、概率质量函数(PMFs)以及贝叶斯假设。

    在Python代码中使用Counter，只需从collections包引入：

    from collections import Counter

    1.判断两个字符串是否由相同的字母集合调换顺序而成的（anagram）

    def is_anagram(word1, word2):
    """Checks whether the words are anagrams.

    word1: string
    word2: string

    returns: boolean
    """

    return Counter(word1) == Counter(word2)

    Counter如果传入的参数是字符串，就会统计字符串中每个字符出现的次数，如果两个字符串由相同的字母集合颠倒顺序而成，则它们Counter的结果应该是一样的。
