  2. virtualenv介绍

    二 01 三月 2016
    By wwt

    virtualenv是一个用来创建独立Python环境的工具。

    什么时候会用到virtualenv？

    1. 假设系统中的两个应用，其中A应用对库LibFoo的版本要求为1，而B应用对同一个库LibFoo的版本要求为2，两个应用对同一个库的要求想冲突了，而系统中库默认都安装到/usr/lib/python2.7/site-packages，那么如何能够同时使用这两个应用呢？

    2. 你的用户权限不够，使得无法将包安装到site-packages目录

    3. 需要使用一个应用，并且与系统其它应用相独立。

    如果你遇到以上的问题，那么virtualenv会对你有帮助。virtualenv可以创建一个环境，拥有着自己的安装目录，而不会与系统的共享库或者其它virtualenv环境相干扰。

    如何安装virtualenv？

    如果系统上已经安装了pip：

    [sudo] pip install virtualenv

    如何使用virtualenv？

    cd my_project_folder

virtual ENV

source bin/activate

deactivate

    以上就是virtualenv的基本使用过程。

    创建一个虚拟环境

    首先，virtual ENV会在当前目录创建名为ENV的虚拟环境，此时在当前目录会自动生成三个目录：bin, include和lib。默认情况下创建的虚拟环境会依赖于系统中的site-packages，即系统中已经安装好的库在虚拟环境中仍然可以使用 ...

    建议50：利用模块设计实现单例模式

    GOF的23种设计模式中，单例是最常用的模式，通过单例模式可以保证系统中一个类只有一个实例而且该实例易于被外界访问，从而方便对实例个数的控制并节约系统资源。

    Python语言实现实例没有那么方便，因为缺乏声明私有构造函数的语法元素，实例又带有类型信息。如：

    class _Singleton(object):
    pass
Singleton = _Singleton()
del _Singleton
another = Singleton.__class__()
print type(another)

# 输出 <class '__main__._Singleton'>

    虽然把Singleton的类定义删除了，但是还是可以通过已有实例的__class__属性生成一个新的实例。

    其实模块采用的方法是天然的单例实现方式：

    • 所有变量都会绑定到模块
    • 模块只初始化一次
    • import 机制是线程安全的
